## Open UI / workflow gaps

These are quality-of-life features, not correctness gaps.

- **Bulk operations** — multi-select clusters/hosts for delete or property edit
- **Saved cluster templates** — preset cluster shapes the user can drop in
- **Mobile / narrow-viewport responsive layout** — currently desktop-only
- **Accessibility review** — keyboard navigation, ARIA, focus management,
  contrast (do a real pass; do not just sprinkle aria-labels)
- **Search / filter** — find clusters/domains by name in a large fleet
- **Additive workbook import** — currently replaces fleet; user may want a
  "merge into existing" mode
- **Per-instance workbook generation** — split a multi-instance fleet into
  one workbook per instance for hand-off

## Test-coverage debt

- **No JSDOM / React Testing Library** in `tests/unit/`. UI is currently
  covered by Playwright E2E plus structural unit tests that inspect the
  JSX source. Real component tests would be a meaningful upgrade.
- **E2E gaps for editor workflows.** Clone / undo-redo / diff modal are
  verified by unit + manual smoke. A focused Playwright spec per workflow
  would catch regressions earlier.
- **Snapshot maintenance.** Adding any field to a factory function tends
  to require `npx vitest run tests/snapshot --update`. Expected, not a bug.

## Operating conventions (carry these to the other PC)

These were the auto-memory entries on the originating machine; auto-memory
is local, so they're recorded here too.

- **PR merge style.** `[TRACKING] Theme N` PRs and most feature PRs use
  squash + delete-branch. `gh pr merge <n> --squash --delete-branch`.
- **No internal refs in UI strings.** Don't surface `Plan-N`, `theme-N`,
  or cell addresses (e.g. `D403`) in rendered JSX. They've been scrubbed
  twice already.
- **Ship UI for new models.** Every theme that adds a `createFleet*Config`
  or new model field must ship a UI panel, even when the tracking spec
  didn't list one.
- **Cell-map backfill audit is two-pass.** Proximity heuristic catches
  most; an exhaustive label-match second pass catches the rest (e.g. SSO
  Domain Name in #93 was missed by proximity alone). Always section-
  context verify before shipping.
- **Symmetric fixes.** When a reviewer scopes a fix to new code, surface
  any pre-existing symmetric cases and offer to extend. Default answer
  has been yes.
- **Review cadence.** Retrospective batch review of merged PRs via a
  3-agent spawn (cell-addresses / logic / tests-emit), then a small
  `chore:` follow-up for LOW items.

## Useful commands

```powershell
# Verify the cell map against the pristine workbooks
npm run verify-cell-map

# Verify the HTML artifact is in sync with engine.js + jsx
npm run verify-html-sync

# Rebuild the single-file HTML
npm run build-html

# Unit tests
npx vitest run

# Snapshot tests (regenerate when factory shapes change)
npx vitest run tests/snapshot --update

# Playwright E2E
npx playwright test

# Open the latest PR's CI checks
gh pr checks <n>

# Cell-map gap audit (proximity heuristic)
node scripts/audit-cell-map-gaps.mjs
```

## When picking the next item

Workbook gaps are independent and small — good warm-up tasks. UI/workflow
gaps generally need design thought first (bulk ops in particular — clarify
the selection model before coding). Test-coverage debt is the highest-
leverage long-term investment but the lowest-velocity in the short term.
